### **🛠️ Troubleshooting & 기술 회고:**
 
1. **문제 1: 5번 메뉴에서 total_payment = 0 으로 변경할 때**
    * **원인:** total_payment를 0으로 다시 만들었는데, 다음 사람이 결제를 진행할 때, 변수가 0으로 초기화가 안되고 이전 사람의 결제금액까지 추가로 결제가 되는 문제가 발생
    * **해결:** total_payment를 total_paymeny로 변수를 잘못 작성하는 문제가 발생하여 문제가 생겼고, 변수를 제대로 작성하여 문제를 해결했습니다.
2. **문제 2: continue 사용시 빨간색 언더바 생성**
    * **원인:** continue를 사용하는데 계속 빨간색 언더바가 생성이 되어 코드가 오류가 발생함
    * **해결:** for 문에 continue를 사용해야 하는데 들여쓰기를 잘못 사용하여 for 문 밖에 continue를 사용하게 되었고, for문 안에 다시 넣어 문제를 해결했습니다.

### **🤖 AI 파트너십 과정**
 1. **내용 1** 장바구니가 초기화 되지 않는 문제
    * **프롬프트 요약:**  ... 첫번째 사람이 라면을 담고 결제를 했어. 그 다음 두번째 사람이 핫도그를 담고 결재를 하려 했는데, 결제 금액은 핫도그만 포함되어 있지만 카트 리스트를 봤을 때는 첫번째 사람이 담은 라면이 포함되어있는데 왜 이런 결과가 발생하는거야? 내가 5번 함수를 정의할 떄, cart = [] 을 사용해서 장바구니를 비워났는데?
    * **적용 내용:** .... cart를 global 변수로 설정 하지 않았기 떄문입니다. cart가 global 변수로 설정 되어 있지 않기 때문에 5번 메뉴를 실행하는 과정에서만 cart 리스트가 빈 리스트로 바뀌고, 그 나머지 상황에서는 cart 리스트가 빈 리스트로 바뀌지 않습니다. 때문에 cart를 global 변수로 설정하면 해결됩니다.


### **🛠️ Troubleshooting & 기술 회고:**
  1. **문제 1:** ... cart 리스트가 초기화가 안되는 문제 발생
     * **원인:** ... cart 리스트를 global 변수로 설정을 하지 않아 cart 리스트가 빈리스트로 바뀌지 않는 상황이 생겼습니다.
     * **해결:** ... cart를 global 변수로 설정하여 문제를 해결했습니다.
       
  2. **문제 2:** ... 6번 메뉴에서 사용자 이름을 정확히 받았지만, 좌석 번호를 제대로 입력하지 않은 경우에, 사용자가 이용한 PC시간은 줄어들지만 좌석을 제대로 반납이 되지 않는 문제가 발생
     * **원인:** ... 사용자 이름을 받으면 바로 시간을 차감하고 난 뒤, 좌석번호를 받아 좌석을 반납하는 순서였기 때문에 위와 같은 문제가 발생했습니다.
     * **해결:** ...  사용자 이름을 받은 후 사용자 리스트에 있는 이름이면 다시 한면 좌석을 선택하는 if 중첩문을 이용하여 문제를 해결했습니다.
    
   3. **문제 3:** ... 2번, 음식을 주문 하는 메뉴에서 for i in range(6) 를 이용하여 음식을 추가하고 제거하였는데, 만약 10번 주문하고 싶은 사용자가 있을 경우, 다시 한번 2번 메뉴에 들어가야하는 번거로움이 발생
      * **원인:** ... 6번만 반복하는 for 구문을 사용했기 때문에 위와 같은 문제가 발생했습니다.

      * **해결:** ... for 구문이 아닌 while True 구문을 이용하여 사용자가 0번을 눌러 주문을 종료할 때 까지 음식을 주문 할 수 있는 코드를 작성하여 문제를 해결했습니다.

    * **프롬프트 요약:**  ... 숫자 변환(int, float)시 try-except가 필수입니다. 또 이름과 같은 문자열 입력 시 try - except가 필요없습니다.
    * **적용 내용:** .... 처음에는 과제에 try - except 개수인 2개만 사용하였는데, 코드를 보다 완벽하게 작성하기 위해서 try - except 구문을 필요로 하는 코드에 모두 적용시켰습니다.
   
    * **내용 2** 만약 밤 10시 이후에 미성년자가 1번 메뉴에서 2시간을 충전한 후, 5번 메뉴에서 미성년자인 것이 발각되었을 때 선택한 좌석, 음식 장바구니, 결제할 금액 이 초기화가 되는데 충전한 2시간은 그대로 남아있는 문제가 발생한다. 결제가 끝나지도 않았는데 시간은 충전되는 문제를 어떻게                      해결하면 좋을까? 방법을 알려줘
    * **프롬프트 요약:**  ... 2가지의 방법이 있습니다. 1번 : 미성년자가 출입을 시도했기 때문에 '야간에 미성년자 출입 시도를 하면 남아있는 기존 시간을 모두 초기화' 하는 우리 PC방만의 규칙을 만들고 코드 적용
                                                     2번 : 충전을 할 때, user_times에 바로 더하는 것이 아닌, temp_time 과 같은 임시 변수를 만들어 보관한 후, 결제가 완료되었을 때, 사용자 데이터에 넣는 방법
    * **적용 내용:** .... 처음에는 2번의 해결방안 처럼 임시변수를 하나 만들어서 코드를 고치려 했지만, AI가 추천한 1번 방법을 보고 난 후, '이러한 방법도 신선한데?' 라는 생각이 들었습니다. 때문에 저는 우리 PC방 만의 새로운 규칙을 만드는 코드를 작성해보았습니다.
    
### **🛠️ Troubleshooting & 기술 회고:**
  1. **문제 1:** ... 5번 메뉴에서 밤 10시가 지났는 지에 대한 질문에서 '아니요'를 입력해야 미성년자임을 판별하기 위해 나이를 묻는 질문이 나와야합니다. 하지만 제가 작성한 코드에서는 '예'를 제외한 아무 글자를 입력하여도 '아니요'로 입력한 것과 같이 프로그램이 실행되는 문제가 발생했습니다.
     * **원인:** ... if - else 구문을 작성할 때, '예', '아니요'를 제외한 값을 입력했을 때, 잘못된 입력임을 판별하는 코드를 넣지 않아서 이러한 문제가 발생했습니다.
     * **해결:** .. if - else 구문 이전에, while True 구문을 이용하여 '예' 또는 '아니요'를 제외한 값을 입력할 시, 다시 제대로 된 값을 입력받는 반복문을 사용하여 문제를 해결했습니다.
    
     *  **문제 2:** ... 9번 메뉴인 사용자 데이터를 저장하는 코드에서 오류가 계속하여 발생했습니다.
     * **원인:** ... with open 구문에서 encoding을 encording 으로 작성하는 실수가 있었기 때문에 오류가 발생하였습니다. 
     * **해결:** .. encording의 잘못된 철자를 encoding으로 제대로 교체하였습니다.
    
     * **문제 3:** ... try - except 구문을 사용할 때, 잘못된 에러 값을 넣는 경우에 다시 메인 메뉴로 돌아가는 번거로운 문제가 발생했습니다.
     * **원인:** ... while True 구문을 사용하지 않았기 때문에 이러한 문제가 발생했습니다.
     * **해결:** .. try - except 구문을 사용하기 전에 while True 구문을 이용하여 에러 값을 입력받는 경우, 메인메뉴로 돌아가는 것이 아닌, 해당 메뉴에서 다시 제대로 된 값을 입력받는 코드를 작성해보았습니다.
