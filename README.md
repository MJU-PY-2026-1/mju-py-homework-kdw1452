## 2. 시나리오 (5~10줄)
* (이곳에 한 학기 동안 개발할 프로그램의 전체 시나리오를 개괄적으로 요약합니다.)
* 이 프로그램은 PC방의 무인 정산 및 주문 시스템입니다.

* 처음으로는 'PC방'을 방문한 손님에게 현재 남은 시간과 충전할 금액을 물어본 후 그 대답을 기록하는 것으로 시스템이 시작됩니다.
* 그 다음으로는 손님이 입력한 금액을 바탕으로 새롭게 추가되는 시간과 총 남은 시간을 정확하게 계산하여 알려줍니다. 현재 진행하는 이벤트로 손님이 10,000원 이상 시간을 추가했을 때에는 감사의 의미로 '서비스 시간 1시간'을 추가해 주는 혜택을 제공할 예정입니다.
* 다음으로는 손님이 주문한 라면, 음료, 커피 등 여러 다양한 먹거리 메뉴를 장바구니에 담는 기능을 넣을 예정입니다.
* PC방의 총 좌석(1~10석) 중 현재 '사용가능한 좌석'을 보여주고 선택하게 합니다.
* 마지막으로 자신이 선택한 좌석, 충전할 시간, 먹거리 목록, 총 결제 금액을 합산하여 화면에 출력할 예정입니다. 
* 또 22시 이후로는 미성년자 출입이 금지되어 있기 때문에, 22시 이후에 결제를 시도하는 손님이 있다면 나이를 물어보고, 미성년자일 경우 결제를 거절하는 기능도 추가할 예정입니다.
* 다음으로는 PC방의 이용을 마쳤을 때, 손님이 앉아있었던 자리를 다시 '사용가능한 좌석'으로 표시한 후, 남은 시간을 메세지로 표시할 예정입니다.

* 마지막으로 특정 비밀번호를 입력하였을 때, PC방의 매출을 표시해 주는 관리자 창을 만들 예정입니다.

## 🟦 [1차 과제: V1.0] 시나리오 기획
    
### **🤖 AI 파트너십 과정**

 1. **내용 1 : 아이디어 생산**
    * **프롬프트 요약:**  현재 나는 파이썬을 해본적 없는 상태이고, 파이썬 입문 프로그램 이라는 교양과목을 수강중인 대학생이야. 이번 학기에 4개의 단계로 발전시킬 프로그램을 만들 예정이야. 혹시 기초적인 내용으로 만들수 있는 프로그램의 주제를 추천해줄 수 있을까?
    * **적용 내용:** AI가 제안해 준 여러 프로그램 중에서 미니 카페 키오스크의 내용이 흥미로웠고, 키오스크를 색다른 매장에 적용시킬 방법이 없을까 고민중에 무인 PC방에 적용을 하게 되었습니다.
      
 2. **내용 2 : 기능 및 메뉴 추천**
    * **프롬프트 요약:** 예상 기능 및 메뉴를 최소 5개를 작성해야 하는데 가능할까?
    * **적용 내용:** 시간충전, 먹거리 주문, 좌석조회, 총 결재 금액 등등 5개의 다양한 메뉴를 추천해 주었고, 이를 바탕으로 예상 기능 및 메뉴를 작성하게 되었습니다. 

 3. **내용 3 : 로드맵 도출**
    * **프롬프트 요약:** 1차 과제 필수 요건이 있는데, 변수 5개, 자료형 3개이상을 충족하기 위해서 로드맵을 도출할 수 있을까?
    * **적용 내용:** 손님 아이디(user_id/str), 충전 금액(recharge_money/int), 추가된 시간(added_time/float), 손님 나이(age/int), 성인 판별여부(adult/bool) 등을 사용하면 변수 5개 이상, 자료형 4개를 사용하게 됩니다.

## 🟩 [2차 과제: V1.0] 입출력 + 리스트 + 조건문 - 향후 작성 예정
### **✨2차 과제 내용:**

  * 이번 과제는 PC방 시간 충전 / 음식메뉴 선택 / 좌석 선택 / 사용자 정보 불러오기 / 최종 결제 / 종료 및 좌석 반납 / 관리자 메뉴 로 이루어져 있으며, 사용자로 부터 이름, 충전할 시간, 주문할 메뉴 종류, 사용할 좌석 데이터를 입력받아 계산을 하거나, 리스트에 넣어 총 결제 금액을 도출해 내는 시스템입니다.
  * input 으로 사용자의 이름, 결재할 시간 등을 입력받아 계산하고, 음식메뉴 와 좌석선택 을 사용자로부터 입력받아 리스트에 저장, 출력하는 기능을 추가했습니다.
  * if 구문을 이용하여 여러 상황에 대해 대처를 하였고, 마지막에는 오늘 하루 총 매출금액을 계산하여 관리자 메뉴에 pw를 입력하면 출력하도록 코드를 작성해보았습니다.
  
### **🤖 AI 파트너십 과정**

1. **내용 1: 전체적인 코드 작성**
    * **프롬프트 요약:** "내가 1~7번 까지의 메뉴를 코드로 작성을 하려고 하는데, 어떻게 코드를 작성해야 할지 감이 안잡혀. 혹시 전체적으로 코드를 작성해 줄 수 있을까? 부족한 부분은 내가 채워볼게."
    * **적용 내용:** AI가 제안한 PC방 시간 충전 부터 마지막 메뉴인 관리자 메뉴까지의 코드를 작성해 주었고, 코드를 하나하나씩 읽고 해석한 후에 문제가 없는지를 확인을 먼저 했습니다. 그 다음으로 메뉴 1개씩 작성을 하여 문제가 없는지 확인을 하였습니다. 

2. **내용 2: 사용자를 리스트에 저장**
    * **프롬프트 요약:** "지난번에 방문한 사용자가 다시 재방문하였을 때, 이전에 사용하고 "남은 시간 + 충전할 시간" 으로 사용자의 PC시간을 출력하고 싶은데 어떻게 하는게 좋을까?"
    * **적용 내용:** user_names 와 user_times 리스트를 만들고 user_names에 없는 이용자라면 리스트에 추가를 하고, 만약 리스트에 존재하는 사용자라면, user_times에 있는 사용자의 시간을 불러오게 하는 코드를 작성해주었습니다.

3. **내용 3: 사용종료시 좌석의 사용가능 여부 변경문제**
    * **프롬프트 요약:** "A가 1번, B가 2번좌석을 선택했고, A가 사용종료를 위해 메뉴 6번에서 자신의 정보를 입력하면 A의 좌석이 1번이 "비어있음"으로 바뀌는게 아니라, 마지막으로 결제한 B의 좌석이 "비어있을"으로 변경되는데, 이 문제를 어떻게 해결할까?"
    * **적용 내용:** 자신의 이름과 자신이 사용한 좌석 번호를 입럭받고 좌석 리스트의 상태를 변경하는 코드를 추가로 작성하여 코드에 넣었습니다. 
### **🛠️ Troubleshooting & 기술 회고:**
 
1. **문제 1: 5번 메뉴에서 total_payment = 0 으로 변경할 때**
    * **원인:** total_payment를 0으로 다시 만들었는데, 다음 사람이 결제를 진행할 때, 변수가 0으로 초기화가 안되고 이전 사람의 결제금액까지 추가로 결제가 되는 문제가 발생
    * **해결:** total_payment를 total_paymeny로 변수를 잘못 작성하는 문제가 발생하여 문제가 생겼고, 변수를 제대로 작성하여 문제를 해결했습니다.
2. **문제 2: continue 사용시 빨간색 언더바 생성**
    * **원인:** continue를 사용하는데 계속 빨간색 언더바가 생성이 되어 코드가 오류가 발생함
    * **해결:** for 문에 continue를 사용해야 하는데 들여쓰기를 잘못 사용하여 for 문 밖에 continue를 사용하게 되었고, for문 안에 다시 넣어 문제를 해결했습니다.
